# Спецификация: Утилита голосового ввода для Windows

## Суть

Системная утилита для Windows. Нажал хоткей → говоришь → текст вставляется порциями в любое активное текстовое поле → нажал снова → остановилась.

---

## Требования

### Активация
- Хоткей: **Right Ctrl** (настраиваемый через конфиг)
- Работает глобально — в любом приложении, без фокуса на окне утилиты

### Распознавание речи
- Движок: **GigaAM v3** (Сбер, `salute-developers/GigaAM` на HuggingFace)
- Локально, без интернета, бесплатно
- Основной язык: русский; английские слова в русской речи — распознаёт нормально
- Обработка порциями по 2–5 секунд (стриминга у модели нет)

### Вставка текста
- Напрямую через симуляцию клавиатуры (pynput или pyautogui)
- **Без буфера обмена** — текст не перезаписывает clipboard
- Вставляется в активное поле в момент распознавания каждой порции

### Визуальная индикация
- Небольшой оверлей на экране (как в Handy / AquaVoice)
- Показывает: идёт запись / обработка / последний распознанный текст
- Не мешает работе с другими окнами (always-on-top, полупрозрачный)

### Автозагрузка
- Опция добавить в автозагрузку Windows
- Через реестр `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` — без прав администратора

### Распространение
- Сборка в `.exe` через PyInstaller
- Один файл — скачал и запустил, Python не нужен
- Целевая платформа: Windows 10+, включая планшеты (Intel Core m3, без GPU)

---

## Почему не подошли существующие решения

| Решение | Проблема |
|---------|---------|
| Handy | Хорошее качество, но текст идёт через буфер обмена — не стриминг |
| AquaVoice | Стриминг есть, но тормозит, перестала выдавать текст порционно |
| Windows Speech | Нет на части устройств (Win10 на планшете), неудобное расположение клавиш |

**Ключевое отличие нашей утилиты:** текст вставляется напрямую, без буфера обмена, с настраиваемым хоткеем.

## Стек

```
Python 3.10+
├── faster-whisper или GigaAM v3  ← распознавание
├── sounddevice / pyaudio          ← захват микрофона
├── pynput                         ← глобальный хоткей + симуляция клавиатуры
├── tkinter или customtkinter      ← оверлей на экране
└── PyInstaller                    ← сборка в .exe
```

---

## Архитектура (поток данных)

```
Микрофон
  → sounddevice (захват аудио буферами)
  → VAD (определение паузы/конца фразы)
  → GigaAM v3 (распознавание порции)
  → pynput.keyboard.type(text)  → активное поле
  → обновить оверлей
```

---

## Конфиг (config.json)

```json
{
  "hotkey": "right_ctrl",
  "model": "gigaam-v3",
  "chunk_duration_sec": 3,
  "autostart": false,
  "overlay_position": "bottom-right"
}
```
